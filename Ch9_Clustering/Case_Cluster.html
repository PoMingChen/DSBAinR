這邊遇到了一個小問題，各個資料的尺度不一致，有數值變數也有類別變數，各個數值的大小、變異和單位也不完全相同，為了讓各個變數對於距離計算的影響較一致，這邊選擇對所有變數做標準化：

  • 類別變數：轉換為1和0
  • 數值變數：轉換至「以最大最小值為1和0的尺度中」

在這邊利用mutate()函數轉換數值變數，清楚且快速達到我們想要做的事

GameTable <- GameTable[,2:7] %>%
  mutate(
    Aft = (Min_Aft - min(Min_Aft)) / (max(Min_Aft)-min(Min_Aft)),
    Eve = (Min_Eve - min(Min_Eve)) / (max(Min_Eve)-min(Min_Eve)),
    Mid = (Min_Mid - min(Min_Mid)) / (max(Min_Mid)-min(Min_Mid)),
    Coin = (Buy_Coin - min(Buy_Coin)) / (max(Buy_Coin)-min(Buy_Coin)),
    Dia = (Buy_Dia - min(Buy_Dia)) / (max(Buy_Dia)-min(Buy_Dia)),
    Car = (Buy_Car - min(Buy_Car)) / (max(Buy_Car)-min(Buy_Car))
  ) %>% cbind(
    GameTable[,c(8,9)]
  )

接著我們選擇用階層式集群分析作為資料探索的方法，利用dist()函數計算距離後，再用hclust()函數進行分析及繪圖。

set.seed(500)
Distance <- dist(GameTable, method = 'euclidean')

hclust(Distance, method = 'complete') %>% plot()

觀察此圖，我們可以發現，玩家大致上可以被分為3到4個群體，因此以下的建模將以這3、4兩種群數分別討論。

資料建模與分析

1. 分為三群

用kmeans()演算法，決定分群數目後、進行分群後，先看看各個群體中的玩家數量

# Implement Kmeans Algorithm:
set.seed(500) # remove the random effect
K <- kmeans(GameTable,3)

ClusterResult <- cbind(
  GameTable,
  K$cluster
) %>% as.data.frame()

colnames(ClusterResult)[ncol(ClusterResult)] <- 'Cluster'

table(ClusterResult$Cluster)
## 
##   1   2   3 
## 253 445 222

接著探討個群體間的連續變數分配，為了將資料轉換成整齊的格式，這邊會用到gather()函數將所有的variables併入同一個欄位裡面

ClusterResultForPlot <- ClusterResult %>%
  gather( key = Continuous_Variable,
          value = Normalized_Value,
          - c(IdentityNovice, IdentityVeteran, Telecomother, Cluster))

ClusterResultForPlot$Continuous_Variable <- ClusterResultForPlot$Continuous_Variable %>% factor( levels = c('Mid','Aft','Eve','Coin','Dia','Car'))

ggplot( data = ClusterResultForPlot) + 
  geom_boxplot( aes( x = Continuous_Variable,
                     y = Normalized_Value),
                size = 0.7) +
  facet_wrap( ~ Cluster)

由上圖看來，這三個群體中，各個特徵的分布還是較為發散，因此我們推論可以再做進一步的分群，來讓各個特徵的分布更為集中，更容易描述。

群體特徵描述 - 連續變數

  • 第一群：上線時間多集中於下午，其他行為較不明顯
  • 第二群：上線時間多集中於晚上，有穩定購買金幣的行為
  • 第三群：上線時間較分散且較長，有穩定購買卡片的行為
  • 第四群：上線時間集中於半夜，有穩定購買鑽石的行為
ClusterResultForPlot <- ClusterResult %>%
  gather( key = Continuous_Variable,
          value = Normalized_Value,
          - c(IdentityNovice, IdentityVeteran, Telecomother, Cluster))

ClusterResultForPlot$Continuous_Variable <- ClusterResultForPlot$Continuous_Variable %>% factor( levels = c('Aft','Eve','Mid','Coin','Dia','Car'))

ggplot( data = ClusterResultForPlot) + 
  geom_boxplot( aes( x = Continuous_Variable,
                     y = Normalized_Value),
                size = 0.7) +
  facet_wrap( ~ Cluster)

群體特徵描述 - 類別變數

  • 玩家身份

新手多屬於第一個群體，並沒有明顯的購買行為；一般玩家多屬於第二群，在晚上的時間上線並有購買金幣的習慣；老手分布在第三和第四群，又以第四群較多，可以說明抽卡和加值鑽石較能吸引老手群體。

針對遊戲營運提出建議

若要增加遊戲內購買，可以針對不同群體提出以下活動：

  • 針對群體一群體一多為新手及學生，較沒有明顯的購買行為，可以透過下午時段限定優惠、學生限定優惠、新手加值優惠等方案，針對群體一進行促銷。

  • 針對群體三群體三整體的上線時間較長，且購買以抽卡為主，建議提供總上線時間的獎勵，提高抽中稀有卡片的機率，以促進群體三的購買行為。

  • 針對群體四群體四的總購買金額最高、上線時間較特別，可以針對該時段設計與其他購買項目的聯合限定促銷，或是獎勵長期玩家的活動，增加群體四的購買金額。

  • 針對全部群體由於各個群體的購買項目皆非常集中，可以嘗試各項目間的聯合促銷活動，擴展其他購買項目的銷售機會
